# Data Models

This document explains the Go domain models in [models.go](../internal/domain/models.go), their business meaning, and how they relate to persistence.

## PreviewMetadata

`PreviewMetadata` represents the social preview content shown when a crawler visits a short URL.

Fields:

- `Title`: preview title, usually from `og:title` or `<title>`.
- `Description`: preview summary, usually from `og:description` or `meta[name=description]`.
- `ImageURL`: preview image, usually from `og:image`.

Persistence: stored on `short_urls` as `preview_title`, `preview_description`, and `preview_image_url`.

Lifecycle: resolved when a short URL is created, refreshed when a crawler visits, and falls back to title `Shared link` plus the original URL when metadata cannot be fetched.

## ShortURL

`ShortURL` is the main aggregate for a shareable link.

Fields:

- `ID`: internal database identifier.
- `Code`: public 7-character base62 short code.
- `OriginalURL`: redirect target.
- `ReferralCode`: default attribution identity for clicks/conversions.
- `Campaign`: default campaign identity.
- `Preview`: cached social preview metadata.
- `CreatedAt`: creation timestamp.
- `UpdatedAt`: last update timestamp, currently used when preview metadata is refreshed.

Persistence: maps to the `short_urls` table.

Important behavior: `Code` is generated randomly and must be unique. Postgres enforces uniqueness; the create usecase retries collisions.

## ClickEvent

`ClickEvent` represents a normal user redirect visit.

Fields:

- `ShortURLID`: database ID of the related short URL.
- `ReferralCode`: referral attributed to this click.
- `Campaign`: campaign attributed to this click.
- `UserAgent`: raw user agent for debugging/classification.
- `Platform`: source platform such as `facebook`, `threads`, `x`, `linkedin`, or `direct`.
- `Device`: classified device such as `desktop`, `mobile`, or `tablet`.
- `Country`: country code from request headers when available.
- `OccurredAt`: click timestamp.

Persistence: maps to `click_events`.

Important behavior: crawler preview visits are not click events. Click recording is best-effort and should not block redirects.

## ConversionEvent

`ConversionEvent` represents a business outcome after a short URL journey.

Fields:

- `ShortURLID`: database ID of the related short URL.
- `ReferralCode`: referral attributed to this conversion.
- `Campaign`: campaign attributed to this conversion.
- `Value`: optional numeric business value, such as order amount or estimated lead value.
- `OccurredAt`: conversion timestamp.

Persistence: maps to `conversion_events`.

Important behavior: when `POST /conversions` omits `referral_code` or `campaign`, the usecase falls back to values stored on the short URL.

## Breakdown

`Breakdown` is a small API response model for grouped analytics.

Fields:

- `Key`: group value, such as referral code, campaign, platform, device, or country.
- `Count`: number of events in that group.

Persistence: not stored directly. It is generated by analytics queries.

## AnalyticsSummary

`AnalyticsSummary` is the response model for `GET /urls/{code}/analytics`.

Fields:

- `Code`: short code being analyzed.
- `TotalClicks`: total redirect click events.
- `TotalConversions`: total conversion events.
- `ClicksByReferral`: click count grouped by referral.
- `ClicksByCampaign`: click count grouped by campaign.
- `ClicksByPlatform`: click count grouped by platform.
- `ClicksByDevice`: click count grouped by device.
- `ClicksByCountry`: click count grouped by country.
- `ConversionsByRef`: conversion count grouped by referral.
- `ConversionsByCamp`: conversion count grouped by campaign.

Persistence: not stored directly. It is assembled from `short_urls`, `click_events`, and `conversion_events`.
